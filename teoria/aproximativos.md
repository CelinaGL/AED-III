# Algoritmos Aproximativos

- [Link mediun](https://ricardomatsumura.medium.com/algoritmos-aproximativos-60109277e16d)

- Abrir mão da qualidade da solução para que o algoritmo termine em um tempo razoável
- Sendo ineficiente encontrar uma solução ótima, nos contentamos em encontrar uma solução boa o suficiente(sub-ótimas)
- Uma forma de obter soluções sub-ótimas é utilizando heurísticas. ver em **Definições**
- Um Algoritmo Aproximativo é uma heurística com alguma garantia na qualidade da solução, expressada por um limite de pior caso para a solução retornada, por exemplo garantindo que a solução retornada é no máximo duas vezes pior que a ótima, não sabendo onde a solução se encontra podendo até ser a ótima
-  Se aplicam a problemas de otimização, em particular os NPD
- "Chamamos um algoritmo de p-aproximado se ele retorna uma solução que é no máximo p vezes pior que a solução ótima e executa em tempo polinomial."
- "Naturalmente, p=1 para problemas NPD só é possível se P=NP."


## Definições

### Heurísticas
- "Uma heurística é um algoritmo eficiente que retorna alguma solução para o problema."
- "Heurísticas podem retornar a melhor solução possível para algumas instâncias, pior possível para outras ou qualquer coisa entre os extremos. Não há garantias sobre a qualidade da solução retornada."
